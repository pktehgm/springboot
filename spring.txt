JavaBean:
- Public no-argument constructor
- Private fields
- Public getters and setters for fields
- implements Serializable
used by frameworks - enables reflection, data binding, introspection
standard way of representing data objects

Reflection - is the ability of a program to inspect and manipulate classes at runtime

Class<?> c = Person.class;
Method m = c.getMethod("setName", String.class);
m.invoke(person, "Alice");

Data binding - Data binding automatically maps data → object fields (or the reverse), uses reflection

person.setName("Alice");
person.setAge(30);

Introspection - a structured form of reflection that analyzes a class according to JavaBean rules

BeanInfo info = Introspector.getBeanInfo(Person.class);
for (PropertyDescriptor pd : info.getPropertyDescriptors()) {
    System.out.println(pd.getName());
}

Reflection → low-level capability
Introspection → structured reflection (JavaBean-focused)
Data Binding → practical use of both

Reflection = can inspect anything
Introspection = understand beans cleanly
Data binding = turn data into objects automatically
