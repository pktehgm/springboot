JavaBean:
- Public no-argument constructor
- Private fields
- Public getters and setters for fields
- implements Serializable
used by frameworks - enables reflection, data binding, introspection
standard way of representing data objects

Reflection - is the ability of a program to inspect and manipulate classes at runtime

Class<?> c = Person.class;
Method m = c.getMethod("setName", String.class);
m.invoke(person, "Alice");

Data binding - Data binding automatically maps data → object fields (or the reverse), uses reflection

person.setName("Alice");
person.setAge(30);

Introspection - a structured form of reflection that analyzes a class according to JavaBean rules

BeanInfo info = Introspector.getBeanInfo(Person.class);
for (PropertyDescriptor pd : info.getPropertyDescriptors()) {
    System.out.println(pd.getName());
}

Reflection → low-level capability
Introspection → structured reflection (JavaBean-focused)
Data Binding → practical use of both

Reflection = can inspect anything
Introspection = understand beans cleanly
Data binding = turn data into objects automatically

Build Java REST API w/ Spring Framework, Spring Boot and Hibernate

REST API - Representational State Transfer Application Programming Interface
REST Principles
    - Stateless - each request contains all info needed
    - Client-Server Architecture - separation of concerns
    - Uniform Interface - standard methods (GET, POST, PUT, DELETE)
    - Resource-Based - resources identified by URIs
    - Representation - resources represented in different formats (JSON, XML)
    - HATEOAS - Hypermedia as the Engine of Application State

Spring MVC for REST API
    - @RestController - class is a REST controller
    - @RequestMapping - map HTTP requests to handler methods
    - @GetMapping - handle HTTP GET requests
    - @PostMapping - handle HTTP POST requests
    - @PutMapping - handle HTTP PUT requests
    - @DeleteMapping - handle HTTP DELETE requests
    - @PathVariable - extract variables from URI
    - @RequestBody - bind HTTP request body to Java object
    - @ResponseBody - bind Java object to HTTP response body
    - Exception Handling - @ControllerAdvice, @ExceptionHandler

Data Validation - @Valid, @NotNull, @Size, @Email
